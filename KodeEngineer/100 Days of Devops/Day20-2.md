# Day 20: Configure Nginx + PHP-FPM Using Unix Sock

<img width="826" height="668" alt="image" src="https://github.com/user-attachments/assets/44164c3e-e4ab-497c-af07-6ccf311153c7" />

The Nautilus application development team is planning to launch a new PHP-based application, which they want to deploy on Nautilus infra in Stratos DC. 
The development team had a meeting with the production support team and they have shared some requirements regarding the infrastructure. Below are the requirements they shared:

a. Install nginx on app server 3 , configure it to use port 8094 and its document root should be /var/www/html.

b. Install php-fpm version 8.2 on app server 3, it must use the unix socket /var/run/php-fpm/default.sock (create the parent directories if don't exist).

c. Configure php-fpm and nginx to work together.

d. Once configured correctly, you can test the website using curl http://stapp03:8094/index.php command from jump host.

NOTE: We have copied two files, index.php and info.php, under /var/www/html as part of the PHP-based application setup. Please do not modify these files.

```md
This task involves setting up a web stack (Nginx and PHP-FPM) on a specific server. Based on the requirements in the image, here is a step-by-step guide to completing the configuration on app server 3.
1. Install Nginx and PHP-FPM
First, connect to app server 3 and install the necessary packages. Since the version 8.2 is specified for PHP, you may need to enable a specific repository (like Remi or AppStream) depending on the OS.
bash
# Update repositories and install Nginx and PHP-FPM 8.2
sudo yum install -y nginx php-fpm
2. Configure PHP-FPM
You need to ensure PHP-FPM uses the specified Unix socket and that the parent directory exists.
Create the socket directory:
bash
sudo mkdir -p /var/run/php-fpm
Edit the PHP-FPM pool configuration:
Open the default config file (usually /etc/php-fpm.d/www.conf):
bash
sudo vi /etc/php-fpm.d/www.conf
Update the listen directive:
Find the listen line and change it to:
text
listen = /var/run/php-fpm/default.sock
Ensure listen.owner and listen.group are set to nginx to avoid permission issues.
3. Configure Nginx
You need to set the port to 8094, the root to /var/www/html, and link it to the PHP socket.
Edit the Nginx configuration:
bash
sudo vi /etc/nginx/conf.d/php_app.conf
Add the following configuration:
nginx
server {
    listen 8094;
    server_name localhost;
    root /var/www/html;
    index index.php index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    location ~ \.php$ {
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_pass unix:/var/run/php-fpm/default.sock;
    }
}
4. Start Services and Test
After saving the configurations, start the services and verify the setup.
Start and Enable Services:
bash
sudo systemctl start php-fpm
sudo systemctl start nginx
sudo systemctl enable php-fpm
sudo systemctl enable nginx
Verification:
Go back to the jump host and run the curl command as specified:
bash
curl http://stapp03:8094/index.php
Important Notes:
Permissions: Make sure the user running Nginx has read access to /var/www/html and write access to the PHP socket.
Firewall: Ensure port 8094 is open on the server if a firewall (like firewalld) is active.
Do Not Modify: As noted in the image, do not change the contents of index.php or info.php.
```
